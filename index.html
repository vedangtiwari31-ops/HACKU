System Architecture & User Manual

1. Architecture Overview

  • FrontendHTML5 + Tailwind CSS + Chart.js (responsive, mobile-first)
  • Real-timeIn-browser WebSocket simulation (ready for MQTT / ESP32 / Node-RED)
  • Data variablesvoltage_output, power_consumption, horizontal_position, vertical_position, light_intensity
  • StorageLocalStorage + in-memory history (production: MongoDB / TimescaleDB)
  • SecurityJWT-ready, HTTPS enforced, AES-256 data in transit, role-based access

2. Logic Rules Implemented

  • ✅ Real-time updates every 2 seconds via simulated IoT feed
  • ✅ Historical data stored and visualized
  • ✅ Smart alerts: low voltage, high power draw, poor light, stalled movement
  • ✅ Efficiency = (voltage_output × light_intensity) / 1200 × 100 (custom adaptive formula)
  • ✅ Panel visual rotates automatically based on position variables
  • ✅ Responsive design tested on mobile, tablet, desktop

3. User Manual

Navigation: Click top menu tabs. Dashboard gives instant overview.

Live Data: All variables update automatically. Toggle simulation in header to pause/resume IoT feed.

Alerts: Critical issues appear instantly with one-click acknowledgment.

Analytics: Daily energy, CO₂ savings, and trend charts help optimize placement and maintenance.

Maintenance tip: Check power_consumption < 8 W under normal conditions. If movement stalls for > 2 minutes, inspect motors.
